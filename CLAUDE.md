# CLAUDE.md — EndSem2Prep

## Project Overview

EndSem2Prep is an interactive, 3D page-flipping digital revision book for Semester 2 engineering exam prep (DSA, AFD, FOML, DBMS). It combines a tactile two-page book UI with live demos, syntax-highlighted code, MCQ quizzes, a Convex-backed leaderboard, and a print-export route — all deployed on Vercel.

**Stack:** React 19 · Vite 8 · React Router v7 · Convex (backend/DB) · react-pageflip · Framer Motion · react-syntax-highlighter · Lucide React · Vercel Analytics

## Code Style & Architecture Rules

**Components**
- Functional components only — no class components.
- Export named exports `FooA` / `FooB` for every spread (left/right page pair). Register both in `src/lib/bookPages.jsx`.
- File names: `PascalCase.jsx` for components/pages, `camelCase.js` for data/utilities.
- Props passed down only — no prop drilling beyond 2 levels; use context for deeper needs.

**Styling**
- All layout and visual styles live in `src/styles/`. No inline `style={{}}` objects for layout concerns.
- Use CSS variables for colors, spacing, and neon theme tokens already defined in `book.css`.
- Each subject has a color identity: DSA = red, AFD = blue, FOML = green, DBMS = purple. Use `data-subj` attribute for theme switching.

**Convex (Backend)**
- All DB mutations/queries go in `convex/`. Never call Convex HTTP endpoints directly from components — use generated hooks (`useQuery`, `useMutation` from `convex/react`).
- Never edit `convex/_generated/` — it is auto-generated by `npx convex dev`.
- Schema changes require running `npx convex dev` to regenerate types.

**Post-Action Rules**
- After writing or modifying any `.jsx`/`.js`/`.ts` file, run `npm run lint` and fix all errors before finishing.
- After any Convex schema or function change, confirm `npx convex dev` is running to validate the schema.

## Adding a New Concept Spread

1. Create `src/pages/concepts/<subject>/<TopicName>.jsx` exporting `<TopicName>A` (left page) and `<TopicName>B` (right page).
2. Import and register both in `src/lib/bookPages.jsx` inside the `BOOK_PAGES` array at the correct subject position.
3. Follow the existing spread pattern: left page = concept explanation + `ExamCard`, right page = code/demo + `SyntaxBlock` or `MemoryBox`.
4. Run `npm run lint` before marking done.

## Strict Restrictions

- **No hardcoded secrets or URLs.** All env values via `import.meta.env.VITE_*`.
- **No placeholder code.** Never write `// TODO: implement later` or stub functions — implement fully or do not add.
- **No deprecated libraries.** Do not add or upgrade to any library without checking it supports React 19.
- **Do not edit `convex/_generated/`.** Always auto-generated.
- **No class components.** Functional + hooks only.
- **No new routes** without registering them in `src/main.jsx`.
- **No direct DOM manipulation** (`document.querySelector` etc.) except inside `PrintBook.jsx`'s print handler.
